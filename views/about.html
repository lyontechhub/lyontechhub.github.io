Qui sommes-nous ?

Lyon Tech Hub est un groupe informel constitué des membres des communautés IT/techniques lyonnaises qui souhaitent y participer. Le groupe est auto-géré par les membres les plus actifs.

Les membres sont des passionnés, des experts, des curieux, des professionnels de tous horizons (éditeurs, sociétés industrielles ou commerciales, startups, indépendants, sociétés de service...). La volonté d'apprendre et d'échanger caractérise ces membres..

Initié sous forme d'un calendrier Google et d'une liste de diffussion, nous avons souhaité augmenter sa visibilité pour tous, afin de simplifier la découverte de nouveaux événements et de faciliter les rencontres.

Comment nous contacter ?

  • Sur twitter @lyontechhub
  • Par mail à la liste de discussion : lyontechhub@googlegroups.com

Vous représentez une communauté ou vous êtes aussi un passionné?
Rejoignez-nous!

Présenter-vous/présentez votre communauté sur la liste de discussion, ou mieux venez à des événements, et inscrivez-vous sur la liste si vous souhaitez suivre les discussions.

Pour inscrire votre communauté, vous n'aurez qu'à faire une pull request sur GitHub pour mettre en ligne les informations concernant votre communauté. Si vous ne savez pas faire une pull request sur GitHub, regarder les informations à fournir ci-dessous (prendre exemple sur les fichiers existants) et envoyez-les par mail à la liste. Demandez aussi sur la liste de diffusion l'accès au calendrier Google pour un ou plusieurs comptes Google.
Voici comment faire la pull request:

  • Aller à https://github.com/lyontechhub/lyontechhub.github.io
  • Forker le repository pour préparer la pull request (cf. https://help.github.com/articles/using-pull-requests)
  • Ajouter ou modifier votte communauté dans le fichier data/communities.json
  • Ajouter ou modifier les détails de votte communauté dans le fichier data/[key].json, où key est la valeur utilisée dans le fichier précédent
  • Ajouter ou modifier le logo de votre communauté au format PNG dans le répertoire imgs, nommé en [key].png (taille de 100px de haut au minimum, pas trop volumineuse)
  • Soumettre la pull request, nous nous nous chargerons de mettre en ligne
